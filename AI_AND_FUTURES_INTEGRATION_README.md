# افزودن Funding Rate + Open Interest + Long/Short Ratio + تحلیل احساسات اخبار

## فایل‌های جدید در این بسته
| فایل | کارش چیه؟ | قابل بک‌تست کامل؟ |
|---|---|---|
| `futures_data_fetcher.py` | دریافت Funding Rate / OI / Long-Short از بایننس | Funding Rate: بله ✅ / بقیه: خیر (فقط ۳۰ روز) |
| `news_sentiment.py` | دریافت اخبار + امتیازدهی احساسات با Claude | خیر (بدون آرشیو تاریخی) |
| `live_data_logger.py` | ثبت روزانه/ساعتی OI، Long/Short، احساسات | این خودش داره تاریخچه می‌سازه |
| `test_funding_rate_edge.py` | تست آماری: آیا Funding Rate واقعاً کمک می‌کنه؟ | همین الان قابل اجراست |
| `features.py` (به‌روزشده) | حالا می‌تونه Funding Rate رو هم به فیچرها اضافه کنه | — |
| `config.py` (به‌روزشده) | یک خط جدید: `ANTHROPIC_API_KEY` | — |

## قدم ۱: کلید API کلود رو تنظیم کن
از https://console.anthropic.com/settings/keys یک کلید بگیر و در `config.py`
جای `PUT_YOUR_ANTHROPIC_API_KEY_HERE` بذار.

⚠️ توجه: استفاده از API کلود هزینه‌بر است (بر اساس تعداد توکن). برای تست
اولیه هزینه‌ش ناچیزه، ولی اگه `live_data_logger.py` رو هر ساعت به‌صورت
دائم اجرا کنی، در طول ماه هزینه‌ی کمی جمع می‌شه - از قیمت‌گذاری فعلی
Anthropic مطلع باش.

## قدم ۲: تست کن آیا Funding Rate واقعاً کمک می‌کنه (فوری، امروز جواب می‌گیری)
```bash
python test_funding_rate_edge.py
```
این دقیقاً مثل `edge_diagnostic.py` قبلی، ولی AUC مدل پایه رو با AUC
مدل «پایه + Funding Rate» مقایسه می‌کنه. اگه بهبود واقعی دید (≥۰.۰۳ واحد AUC)،
یعنی این فیچر ارزش اضافه‌کردن داره. **خروجی رو برام بفرست تا با هم تفسیرش کنیم.**

## قدم ۳: شروع ثبت زنده‌ی OI / Long-Short / احساسات (از همین امروز)
```bash
python live_data_logger.py
```
هر بار که اجراش کنی، یک ردیف جدید به `live_enriched_data_log.csv` اضافه
می‌شه. برای اینکه به‌صورت خودکار هر ساعت اجرا بشه، می‌تونی:
- در ویندوز: از **Task Scheduler** استفاده کنی (هر ۱ ساعت این اسکریپت رو اجرا کنه)
- یا یک حلقه‌ی ساده با `schedule` بسازیم که به‌صورت مداوم در پس‌زمینه اجرا بشه (بگو اگه این رو می‌خوای، براش کد می‌نویسم)

بعد از ۲-۴ هفته که چند صد ردیف جمع شد، می‌تونیم با همون روش آماری قبلی
(AUC، bootstrap، permutation test) بررسی کنیم آیا این داده‌ها واقعاً
سیگنال دارن یا نه - **دقیقاً با همون صداقت آماری که تا الان داشتیم**، نه
فرض کورکورانه که «چون هوش مصنوعیه پس حتما بهتره».

## یک یادآوری مهم و صادقانه
حتی Claude API هم فقط می‌تونه **از اطلاعاتی که بهش می‌دیم** (عناوین خبری)
یک امتیاز احساسات منطقی دربیاره - این با «پیش‌بینی جادویی بازار» فرق داره.
تحلیل احساسات اخبار یک فیچر معقول و رایج در صنعته، ولی نه معجزه؛ باید مثل
بقیه‌ی فیچرها با آمار تست بشه، نه فقط چون "هوش مصنوعی" توش هست باور بشه.
